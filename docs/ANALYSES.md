# How to add an analysis

Copy the template, write the script, register a task, open a PR. Same
pattern for every contributor.

## In 60 seconds

```bash
cp -r analyses/_template analyses/yourname
$EDITOR analyses/yourname/main.py
# add an entry under [tasks."analysis:yourname"] in mise.toml
mise run analysis:yourname
```

## Folder layout for one analysis

```
analyses/<yourname>/
├── README.md         what this answers, what it writes, ≤300 words
├── main.py           entry point
├── report.md         human-readable summary (auto-generated by main.py)
├── summary.json      machine-readable headline numbers (auto-generated)
├── *.csv             any derived tables this analysis produces
└── scratch/          gitignored playground (notebooks, scratch CSVs)
```

The only mandatory files are `README.md` and `main.py`. Everything else
is convention.

## The script contract

Every `main.py` should:

1. **Load via `load_designs()`.** Never hard-code a path.
2. **Compute deterministically.** No HTTP calls, no fresh database
   queries, no GPU — those happen upstream when the canonical CSV is
   regenerated.
3. **Write outputs into your own folder.** A `report.md`, a
   `summary.json`, any tables. Don't touch any other folder.
4. **Be idempotent.** Running `main.py` twice produces the same output.

Skeleton:

```python
"""
What this analysis answers, in one sentence.

Outputs:
- analyses/yourname/report.md
- analyses/yourname/summary.json
"""
from __future__ import annotations

import json
from pathlib import Path

from scripts.utils import load_designs

OUT = Path(__file__).resolve().parent


def main() -> None:
    df = load_designs(only_screened=True)
    summary = {"n_hits": int(df.is_hit.sum())}
    (OUT / "summary.json").write_text(json.dumps(summary, indent=2))
    (OUT / "report.md").write_text("# Headline\n\n" + json.dumps(summary, indent=2))


if __name__ == "__main__":
    main()
```

## Registering with `mise`

Open `mise.toml`, add:

```toml
[tasks."analysis:yourname"]
description = "One-sentence answer to: what does this analysis tell us?"
run = "uv run python analyses/yourname/main.py"
```

Then add `"analysis:yourname"` to the `depends = [...]` list of
`[tasks."analysis:all"]` if it's load-bearing for the paper.

If you don't want `mise`, drop the same target into the Makefile.

## Notebooks vs scripts

Both are fine, but:

- **A `main.py` is required.** Notebooks alone don't run in CI. Develop
  in a notebook if you want, but ship a `main.py` that reproduces the
  result.
- **Load-bearing notebooks** live in `analyses/yourname/notebooks/` and
  can be committed. Exploratory notebooks live in
  `analyses/yourname/scratch/` (gitignored).

## What goes in `report.md`

≤1 page. The structure that's worked best:

```markdown
# <Analysis name>

**Headline:** one-sentence finding with a number.

## Method

What was computed, what filter was applied, what stat test was used.
Cite the column names from `data/designs.csv` so the reader can re-derive.

## Results

A small table or two. Effect size + 95% CI + p-value. Per-cohort split
if relevant. No prose victories.

## Caveats

The N is small / the metric is noisy / the cohort is unbalanced. Always
include this section. Every analysis here has caveats.
```

## What goes in `summary.json`

Machine-readable, no prose. Future tools (paper auto-population,
dashboard, LLM cross-checks) read this.

```json
{
  "n_designs": 141,
  "n_screened": 100,
  "hit_rate_overall": 0.37,
  "hit_rate_human": 0.385,
  "hit_rate_agent": 0.343,
  "fisher_p": 0.83,
  "fisher_odds_ratio": 1.20
}
```

## Reviewing yourself before the PR

- `mise run analysis:yourname` succeeds in a fresh checkout.
- `mise run analysis:all` still passes (you didn't break anyone else).
- `report.md` is ≤1 page and has a Caveats section.
- `summary.json` is valid JSON.
- Any figure was saved via `scripts.plotting.save_figure()` into
  `figures/paper/` or `figures/exploration/`.
- `ruff check analyses/yourname/` passes.

## Common pitfalls

1. **Reading `data/raw_lab/bli_results.csv` for KDs.** Use
   `kd_arith_mean_nM_all` from `designs.csv` — replicate filtering is
   already applied.
2. **Inferring cohort from team name.** `claude-sonnet-4.6` is an agent,
   `Marcel` is a human. Use `is_human`.
3. **Reporting p-values without effect sizes.** Always pair them.
4. **Reporting hit rates without the denominator.** "Agent hit rate is
   X%" is meaningless without "(N=35 screened)" next to it.
5. **Plotting without the theme.** Call `apply_theme()` first; pick
   colours from `theme/palettes.json`.
6. **Writing the CSV in place.** Don't. Make a derived file in your own
   folder.

## The four canonical analyses

Already wired up, useful to read for inspiration:

- `analyses/human_vs_agent/` — cohort stats (Fisher, MW, Spearman, χ²).
  The most-cited analysis for the paper.
- `analyses/leaderboard/` — top-N tables driving the figure-1
  leaderboard.
- `analyses/sequence_diversity/` — within-team identity + cohort k-mer
  entropy.
- `analyses/methods/` — design-method × outcome cross-tabs.

Copy whichever is closest to what you're trying to do.
