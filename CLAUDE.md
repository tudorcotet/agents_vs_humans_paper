# Claude Code instructions — agents_vs_humans_paper

## Read first

[`docs/DATA.md`](docs/DATA.md) for the canonical 141-row table. Then
[`CONTRIBUTING.md`](CONTRIBUTING.md) for the collaboration rules. Then
[`docs/STYLE_GUIDE.md`](docs/STYLE_GUIDE.md) before writing prose.

## What this repo is

Data, code, figures, and manuscript source for the paper coming out of a
one-day hackathon (Feb 28 2026, SF) where **10 human teams (81 designs)
and 6 autonomous AI agent teams (60 designs)** designed binders against
TREM2. Top 100 by Boltz-2 ipSAE went to the BLI assay. Wet-lab outcome:
37 binders / 11 non-expressing / 52 non-binders.

**The agent-vs-human framing is the entire point of the paper.** Never
collapse the cohorts. Every hit rate, KD distribution, sequence-diversity
number must split by `is_human`.

## Contracts

1. **Source of truth:** `data/designs.csv` (and the parquet sibling).
   141 rows. Load via `from scripts.utils import load_designs`.
2. **Per-replicate access:** `load_replicates()`, then filter
   `selected & ~excluded & fixed`.
3. **Per-collaborator subdirs:** `analyses/<name>/`. Writes only into
   that subdir. Copy `analyses/_template/` to start.
4. **Plotting:** `scripts.plotting.apply_theme()` at the top of every
   figure script. Colours from `theme/palettes.json`. Save with
   `scripts.plotting.save_figure()` into `figures/paper/`.
5. **Blog figures:** the 7 hand-authored HTML files live in
   `figures/blog/`. They are NOT regenerated by any Python script.
   Update them by hand when the underlying numbers change.

## When asked to add an analysis

1. `cp -r analyses/_template analyses/<name>/`.
2. Edit `main.py`. Write `report.md` (human) and `summary.json` (machine)
   next to it.
3. Register a task in `mise.toml` matching the existing entries.
4. If load-bearing for the paper, add it to `README.md`'s list.

## Style — plotting and typography

- **Palette:** `theme/palettes.json`. For the agent-vs-human split use
  the `agent_vs_human` entry. Same two hues on every figure.
- **Fonts:** GT Pressura Extended (display), Geist (body), Geist Mono /
  IBM Plex Mono (numbers). Install via `theme/fonts/README.md`. If
  missing, `apply_theme()` falls back to DejaVu Sans cleanly.
- **No 3D, no rainbow.** Sequential data → viridis. Categorical → palette.

## Voice

- **Paper:** clinical, measured, methods-paper voice. Numbers where you'd
  reach for an adjective.
- **Blog / social copy** lives elsewhere and is written separately; this
  repo is for the paper.

## Numbers come from disk, not from the model

- Don't fabricate KDs. Pull from `kd_arith_mean_nM_all` /
  `pkd_arith_mean`. Cite the column.
- Don't fabricate ipSAE / pTM / pLDDT. Use `submitted_ipsae` and the
  `boltz2_*` columns; 136/141 designs are populated, the 5 nulls are
  bottom-ranked non-screened designs.
- Don't infer team labels from sequence patterns. Use `team`.
- Don't infer cohort from team names. Use `is_human`.

## Stats conventions

- Affinity: `pkd = -log10(kd_M)`. Higher = tighter. Non-binders are null.
- Replicates: arithmetic mean of `kd_nM` over `selected & ~excluded & fixed`.
  Already applied in `designs.csv`.
- Correlations: Spearman ρ + bootstrap 95% CI (1000 iters, percentile).
- Distributions: Mann-Whitney U two-sided; Kruskal-Wallis for >2 groups.
- 2×2 hit rate: Fisher's exact two-sided. Print the raw counts.
- Multiple testing: BH-FDR at q=0.10 when running >5 tests. State the threshold.
- **Always pair p-value with effect size.** No exceptions.

## Domain notes

- **Target:** human TREM2 ectodomain construct (Acro `TR2-H52H5`, 175 aa,
  19.3 kDa): initiator Met + residues 19–174 of UniProt Q9NZC2 +
  GGGSGGGS linker + 10×His. The exact sequence is in
  `data/target/trem2_construct.fasta`. Designers fold against this — not
  the bare IgSF domain.
- **Positive controls:** AL002 (Alector, failed Phase 2 in 2024) and
  VHB937 (Novartis, ongoing Phase 2). Sanity check, not a "must beat" bar.
- **"Agent"** means end-to-end autonomous on the bioArena platform with
  tool access (Boltz, ipSAE, …), **not** "raw LM samples sequences."
  Don't elide.
- **The 6 agent teams:** claude-sonnet-4.6, qwen-3.5-plus,
  grok-4.1-fast, Gemini 3.1 Pro, GLM 5, GPT 5.2.
- **MRAZS used Mosaic.** MRAZS ≠ Escalante Bio. Mosaic is a JAX
  composite-objective wrapper maintained by Escalante. Don't conflate.
- **`Foundry` method label** = "the agent picked its own tools." Bucket
  as `Other` family. Not a single algorithm.
- **Top-100 cherry-pick:** 41 designs were never screened. Filter on
  `submitted_to_lab` and state your denominator explicitly.

## What NOT to do

- Don't re-implement the canonical join. `load_designs()` does it.
- Don't read `data/raw_lab/bli_results.csv` for headline KDs — the
  replicate filter is already applied in `designs.csv`.
- Don't add scratch files outside `analyses/<name>/scratch/`.
- Don't add a new top-level directory without flagging it in the PR.
